# In-Memory Stage Orchestration Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Refactor the Paper2Code execution stages to expose standard programmatic `run_stage(config)` interfaces and wire them into an in-memory orchestrator in `pipeline.py`.

**Architecture:** We wrap the existing top-level execution code of all stage scripts in a unified `run_stage(config)` function that uses duck typing (`getattr`) to retrieve properties cleanly. Standalone execution will parse arguments and invoke `run_stage` under a `if __name__ == "__main__":` block. `pipeline.py` will then import these stage functions dynamically via `importlib.import_module` (avoiding Python identifier syntax limitations for filenames starting with digits) and execute them in-memory, completely removing subprocess latency and process handling code.

**Tech Stack:** Python 3.13, SQLModel, pytest

```python
import importlib

def load_stage_module(module_name: str):
    """Import a stage module programmatically to support prefix paths and digit filenames."""
    for prefix in ["codes.", ""]:
        try:
            return importlib.import_module(prefix + module_name)
        except ImportError:
            continue
    raise ImportError(f"Could not import stage module: {module_name}")

def run_pipeline(config: PipelineConfig) -> PipelineResult:
    # ... Database initialization ...
    
    # 2. Run planning stage

        current_stage = "planning"
        try:
            mod = load_stage_module("1_planning")
            mod.run_stage(config)
            completed_stages.append(current_stage)
        except Exception as e:
            # handle failures cleanly...
```
