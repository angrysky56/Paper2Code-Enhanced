# Global CLI Installation Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Configure package metadata in `pyproject.toml` to register `paper2code` as a global command-line entrypoint and verify its installation.

**Architecture:** We append the `[project.scripts]` key to `pyproject.toml` referencing `codes.pipeline:main` as the handler. Using `uv pip install -e .`, the script is linked to the active environment's binary path, allowing typing `paper2code --help` anywhere within the virtual environment.

**Tech Stack:** Python 3.13, uv, pyproject.toml

---

### Task 1: Register Entrypoint in `pyproject.toml`

**Files:**
- Modify: `pyproject.toml`
- Test: Verify with `paper2code --help`

**Step 1: Write the failing test**

Run a command to verify that `paper2code` is not currently an executable on the path:
Run: `paper2code --help`
Expected: Command not found or shell error.

**Step 2: Write minimal implementation**

Edit `pyproject.toml` to append:

```toml
[project.scripts]
paper2code = "codes.pipeline:main"
```

Also, to ensure standard packaging tools can resolve the code layout properly:
Ensure `codes/__init__.py` exists.

**Step 3: Run installation command**

Run: `uv pip install -e .`

**Step 4: Run test to verify it passes**

Run: `uv run paper2code --help`
Expected: The CLI help message displays successfully.

**Step 5: Commit**

```bash
git add pyproject.toml
git commit -m "feat(cli): register paper2code global command in pyproject.toml"
```
